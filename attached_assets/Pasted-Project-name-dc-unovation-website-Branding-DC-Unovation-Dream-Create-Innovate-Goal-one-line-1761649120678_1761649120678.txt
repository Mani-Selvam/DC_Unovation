3) Forms → n8n webhook mapping (endpoints to call)

Use these exact webhook paths (POST JSON). For local dev assume n8n running at http://localhost:5678/webhook — the agent should call full URLs like http://localhost:5678/webhook/<path>.

lead_cta_tracking — POST when hero CTA clicked (if no form shown)
Payload: { page, action, name?, email?, timestamp }

service_inquiry_form — POST for service quote form
Payload: { name, email, service, message, page, timestamp }

project_interest_tracking — POST from portfolio modal (project + user details)
Payload: { project, name, email, message?, page, timestamp }

feedback_submission — POST from feedback/testimonial form
Payload: { name, email, rating, message, timestamp }

newsletter_subscription — POST from blog signup (if separate)
Payload: { email, name?, source, timestamp }

job_applications_submission — POST multipart (file) from job applications (for dev, file can upload to placeholder endpoint; include resumeUrl in payload if file storage implemented later)
Payload: { name, email, phone, role, message, resumeFileName?, timestamp }

context_form_submission — POST from main contact page (same as contact above)
Payload: { name, email, phone?, message, page, timestamp }

newsletter_footer_signup — POST from footer small form
Payload: { email, source: 'footer', timestamp }

NOTE: For local dev, these requests can use fetch to http://localhost:5678/webhook/<path>. Add an easy-to-edit ENV constant in .env.local named VITE_N8N_BASE_URL (default http://localhost:5678/webhook).

10) Example fetch code (copy into project)
// utils/api.ts
const base = import.meta.env.VITE_N8N_BASE_URL || 'http://localhost:5678/webhook';

export async function postToWebhook(path: string, data: any) {
  const res = await fetch(`${base}/${path}`, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ ...data, timestamp: new Date().toISOString() })
  });
  return res;
}
